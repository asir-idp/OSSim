Example: Contiguous memory management with variable-sized partitions (Swap)

The first memory is not partitioned, partitions are created as processes are loaded and have the same size of these, later partitions are free (when a process completes, for example) and are occupied by new processes, if a process is smaller than a partition, this is divided into two parts, the first one as large as the process and the other the remainder.

Memory has a finite size, only a limited number of processes can be in memory at once, but the swap space (backing store) allows to keep some of these processes, generally the less active, and leave available memory for the new ones. Swapped out processes are allocated back into memory when needed.


Process Size:
Units
P1 5

Processes are allocated in the table order.

Comments:

  • There is no internal fragmentation, partitions are always of the size of the process
  • The last two processes ("P7" and "P8") can not be loaded because memory is full
  • If some process is swapped out) that will free up enough space to allocate. For example "P2" and "P4" (right click on the process, pop up menu)
  • To allocate back processes that are swapped must make room in memory, for example swapping "P6". Then "P2" and "P4" may be swapped in (right click on memory, pop up menu)
