Ejercicio. Acceso indexado, UNIX

Enunciado.

La memoria secundaria tiene un tamaño de 4 x 1024 unidades, el tamaño del bloque es de 1 unidades.

La memoria está inicialmente vacía, los objetos se crean en el orden de la tabla. Las direcciones @ 0 hasta @ 127 están reservadas y no se pueden utilizar. El número de i-nodo se asigna secuencialmente desde 0 (root).

Dada la tabla de objetos del sistema de ficheros del ejercicio, responde a las preguntas:

  1. Indica el tamaño total en bloques (datos + indirectos) de los diferentes objetos y el número de i-nodo:
    Objeto Tamaño
    (En bloques)    		 I-nodo
    / 0
    folder1
    file1.txt
    file2.txt
    link1
    folder2
    file3.txt
    folder3
    file4.txt
    link2
  2. ¿Cuál es el tamaño máximo de un archivo en este esquema?

Datos.

Árbol tipo Tamaño
(Unidades)
/ Directorio 1
folder1 Directorio 1
file1.txt Archivo 101
file2.txt Archivo 200
link1 Enlace fuerte
(A file1.txt)		 1
folder2 Directorio 1
file3.txt Archivo 460
folder3 Directorio 1
file4.txt Archivo 120

(A link1)		 0
