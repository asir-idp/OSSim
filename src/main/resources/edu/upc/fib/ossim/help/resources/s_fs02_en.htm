Example: Linked Allocation with FAT (block size: 4 units)

In this example a folder tree is created with different types of file system objects: folders, files and links.

Reminder 1: the folders' color is green and the links are blue. The files color is defined when created.

Reminder 2: In the simulation the folder size is 1 block, while the links do not occupy disk space, they are simply entries in its folder data.


Tree Type Size
Units		 Blocks
C: Folder 1 1
folder1 Folder 1 1
file1.txt File 101 26
file2.txt File 200 50
folder2 Folder 1 1
file3.txt File 35 9.
Folder3 Folder 1 1
file4.txt File 120 30
link1 Soft link
(to file1.txt)		 0 0

Block size is 4 units.

Comments:

  • The last block of files "file1.txt" and "file3.txt" is not full, but the extra space can not be exploited, entire blocks are used
  • (Compared with Example 1) Folders contain the same information but take up much more space, in this example 4 units
