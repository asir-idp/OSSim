Example: Linked Allocation with FAT. File System objects

In this example a folder tree is created with different types of file system objects: folders, files and links.

Reminder 1: the folders' color is green and the links are blue. The files color is defined when created.

Reminder 2: In the simulation the folder size is 1 block, while the links do not occupy disk space, they are simply entries in its folder data.


Tree Type Size
Units
C: Folder 1
folder1 Folder 1
file1.txt File 101
file2.txt File 200
folder2 Folder 1
file3.txt File 35
Folder3 Folder 1
file4.txt File 120
link1 Soft link
(to file1.txt)		 0

Block size is 1 unit.

Comments:

  • File system root is "C:" and path separator is "\"
  • In each new folder, two links are automatically created "." and ".." pointing to the directory itself and its parent respectively
  • First block of the different objects corresponds with the first entry in its FAT table
  • If a file referenced by a link is deleted (link1 -> file1.txt), link remains but become unusable
  • If a folder is deleted, all the objects hanging from it are also deleted
